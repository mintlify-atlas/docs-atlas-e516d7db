---
title: 'List tasks'
description: 'Retrieve a paginated, filterable list of tasks.'
---

`GET /api/tasks`

Returns a paginated list of tasks. Regular users see only their own tasks. Admin users can see all tasks and filter by `user_id`.

<Info>**Authentication:** Bearer access token required.</Info>

## Query parameters

<ParamField query="page" type="integer" default="1">
  Page number to retrieve.
</ParamField>

<ParamField query="per_page" type="integer" default="10">
  Number of tasks per page.
</ParamField>

<ParamField query="sort_by" type="string" default="created_at">
  Field to sort by. Allowed values: `id`, `title`, `status`, `priority`, `due_date`, `created_at`, `updated_at`.
</ParamField>

<ParamField query="sort_order" type="string" default="desc">
  Sort direction. Allowed values: `asc`, `desc`.
</ParamField>

<ParamField query="status" type="string">
  Filter by task status. Allowed values: `pending`, `in_progress`, `completed`, `cancelled`.
</ParamField>

<ParamField query="priority" type="string">
  Filter by task priority. Allowed values: `low`, `medium`, `high`, `urgent`.
</ParamField>

<ParamField query="tag_id" type="integer">
  Filter by tag ID.
</ParamField>

<ParamField query="tag_name" type="string">
  Filter by tag name (case-insensitive substring match).
</ParamField>

<ParamField query="search" type="string">
  Search tasks by title or description (case-insensitive).
</ParamField>

<ParamField query="overdue" type="boolean">
  When `true`, returns only tasks whose due date has passed and that are not completed.
</ParamField>

<ParamField query="due_date_from" type="string">
  Filter tasks with a due date on or after this ISO 8601 timestamp (e.g., `2024-01-01T00:00:00`).
</ParamField>

<ParamField query="due_date_to" type="string">
  Filter tasks with a due date on or before this ISO 8601 timestamp (e.g., `2024-12-31T23:59:59`).
</ParamField>

<ParamField query="user_id" type="integer">
  Filter tasks by owner. **Admin only.**
</ParamField>
