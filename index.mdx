## What is TaskForge API?

TaskForge API is a production-grade RESTful backend for task management. It is built with **Flask 3**, **SQLAlchemy 2**, and deployed on **Azure App Service** with **Azure SQL Database**. The API provides a complete foundation for any task-tracking application, featuring secure authentication, fine-grained access control, and rich query capabilities.

**Live demo:** [https://task-forge-gbd6h8gtg8hchve9.chilecentral-01.azurewebsites.net/api/docs](https://task-forge-gbd6h8gtg8hchve9.chilecentral-01.azurewebsites.net/api/docs)

## API overview

All endpoints are prefixed with `/api`. Authentication uses Bearer tokens in the `Authorization` header.

```
Base URL: https://task-forge-gbd6h8gtg8hchve9.chilecentral-01.azurewebsites.net
Auth:     Authorization: Bearer <access_token>
```

| Resource | Endpoints |
|---|---|
| Auth | `POST /api/auth/register`, `POST /api/auth/login`, `POST /api/auth/refresh`, `POST /api/auth/logout` |
| Tasks | `GET/POST /api/tasks`, `GET/PUT/DELETE /api/tasks/{id}`, `GET /api/tasks/statistics`, `GET /api/tasks/export` |
| Tags | `GET/POST /api/tags`, `GET/PUT/DELETE /api/tags/{id}` |
| Users | `GET /api/users`, `GET/PUT/DELETE /api/users/{id}`, `POST /api/users/{id}/activate` |
| Health | `GET /api/health` |

<Note>
  All API responses follow a consistent envelope format: `{ "success": true, "data": {...} }` for success and `{ "success": false, "message": "..." }` for errors.
</Note>
