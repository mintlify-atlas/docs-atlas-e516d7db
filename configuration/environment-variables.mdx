TaskForge API is configured entirely through environment variables. This page lists every variable, its default value, and whether it is required.

## Create your .env file

Copy the example file and edit it with your own values:

```bash
cp .env.example .env
```

<Warning>
  Never commit your `.env` file to source control. It is already listed in `.gitignore`, but double-check before pushing.
</Warning>

## Complete .env example

```env
# Application
FLASK_APP=run.py
FLASK_ENV=development
SECRET_KEY=your-secret-key-here-change-in-production
JWT_SECRET_KEY=your-jwt-secret-key-here-change-in-production
APP_NAME=TaskForge API
APP_VERSION=1.0.0

# Azure SQL Database
AZURE_SQL_SERVER=yourserver.database.windows.net
AZURE_SQL_DATABASE=taskforge_db
AZURE_SQL_USER=yourusername
AZURE_SQL_PASSWORD=YourPassword123
AZURE_SQL_PORT=1433

# JWT
JWT_ACCESS_TOKEN_EXPIRES=3600
JWT_REFRESH_TOKEN_EXPIRES=2592000

# Rate limiting
RATELIMIT_ENABLED=true
RATELIMIT_STORAGE_URL=memory://
RATELIMIT_DEFAULT=200 per day;50 per hour

# CORS
CORS_ORIGINS=http://localhost:3000,http://localhost:5173

# Pagination
DEFAULT_PAGE_SIZE=10
MAX_PAGE_SIZE=100

# Logging
LOG_LEVEL=INFO
LOG_FORMAT=json
```

## Variables by category

### Application

| Variable | Description | Default | Required |
|---|---|---|---|
| `FLASK_APP` | Entry point module for Flask | `run.py` | No |
| `FLASK_ENV` | Runtime environment (`development`, `production`, `testing`) | `development` | No |
| `SECRET_KEY` | Flask session and CSRF secret | `dev-secret-key-change-in-production` | **Yes** |
| `APP_NAME` | Application name returned in API metadata | `TaskForge API` | No |
| `APP_VERSION` | Application version returned in API metadata | `1.0.0` | No |

<Warning>
  Set `SECRET_KEY` to a long, random string in production. Never use the default value. Generate one with:

  ```bash
  python -c "import secrets; print(secrets.token_hex(32))"
  ```
</Warning>

### Database

| Variable | Description | Default | Required |
|---|---|---|---|
| `AZURE_SQL_SERVER` | Azure SQL server hostname | — | No* |
| `AZURE_SQL_DATABASE` | Database name | `taskforge_db` | No |
| `AZURE_SQL_USER` | Database username | — | No* |
| `AZURE_SQL_PASSWORD` | Database password | — | No* |
| `AZURE_SQL_PORT` | Database port | `1433` | No |

*When all four Azure SQL variables are set, the API connects to Azure SQL. When any are missing, it falls back to SQLite. See [Database configuration](/configuration/database) for details.

### JWT

| Variable | Description | Default | Required |
|---|---|---|---|
| `JWT_SECRET_KEY` | Secret used to sign JWT tokens | `jwt-secret-key-change-in-production` | **Yes** |
| `JWT_ACCESS_TOKEN_EXPIRES` | Access token lifetime in seconds | `3600` (1 hour) | No |
| `JWT_REFRESH_TOKEN_EXPIRES` | Refresh token lifetime in seconds | `2592000` (30 days) | No |

<Warning>
  Set `JWT_SECRET_KEY` to a separate, unique random string in production — distinct from `SECRET_KEY`. Generate one with:

  ```bash
  python -c "import secrets; print(secrets.token_hex(32))"
  ```
</Warning>

### Rate limiting

| Variable | Description | Default | Required |
|---|---|---|---|
| `RATELIMIT_ENABLED` | Enable or disable rate limiting | `true` | No |
| `RATELIMIT_STORAGE_URL` | Backend for storing rate limit counters | `memory://` | No |
| `RATELIMIT_DEFAULT` | Default limits applied to all endpoints | `200 per day;50 per hour` | No |

See [Rate limiting](/configuration/rate-limiting) for full details.

### CORS

| Variable | Description | Default | Required |
|---|---|---|---|
| `CORS_ORIGINS` | Comma-separated list of allowed origins | `http://localhost:3000,http://localhost:5173` | No |

See [CORS configuration](/configuration/cors) for full details.

### Pagination

| Variable | Description | Default | Required |
|---|---|---|---|
| `DEFAULT_PAGE_SIZE` | Default number of items per page | `10` | No |
| `MAX_PAGE_SIZE` | Maximum number of items per page | `100` | No |

### Logging

| Variable | Description | Default | Required |
|---|---|---|---|
| `LOG_LEVEL` | Log verbosity (`DEBUG`, `INFO`, `WARNING`, `ERROR`) | `INFO` | No |
| `LOG_FORMAT` | Log output format (`json`, `text`) | `json` | No |
