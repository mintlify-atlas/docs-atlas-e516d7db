`GET /api/tasks` accepts a rich set of query parameters for filtering, sorting, and paginating results. You can combine any number of these parameters in a single request.

## Pagination

| Parameter | Type | Default | Description |
|---|---|---|---|
| `page` | integer | `1` | Page number to retrieve |
| `per_page` | integer | `10` | Items per page. Maximum is `100` |

Every list response includes a `pagination` object:

```json
"pagination": {
  "page": 2,
  "per_page": 10,
  "total": 47,
  "total_pages": 5,
  "has_next": true,
  "has_prev": true
}
```

## Sorting

| Parameter | Type | Default | Description |
|---|---|---|---|
| `sort_by` | string | `created_at` | Field to sort by |
| `sort_order` | string | `desc` | Sort direction: `asc` or `desc` |

Supported `sort_by` values for tasks: `id`, `title`, `status`, `priority`, `due_date`, `created_at`, `updated_at`.

If you supply an unsupported `sort_by` value, the API falls back to the first allowed field (`id`).

## Filters

### Status filter

Filter tasks by their current status.

| Value | Description |
|---|---|
| `pending` | Not yet started |
| `in_progress` | Currently in progress |
| `completed` | Finished |
| `cancelled` | Abandoned |

```bash
curl "https://api.example.com/api/tasks?status=in_progress" \
  -H "Authorization: Bearer <access_token>"
```

### Priority filter

Filter tasks by priority level.

| Value | Description |
|---|---|
| `low` | Low urgency |
| `medium` | Normal urgency |
| `high` | High urgency |
| `urgent` | Highest urgency |

```bash
curl "https://api.example.com/api/tasks?priority=urgent" \
  -H "Authorization: Bearer <access_token>"
```

### Tag filters

Filter by a specific tag using either its ID or name.

| Parameter | Type | Description |
|---|---|---|
| `tag_id` | integer | Return tasks that have the tag with this ID |
| `tag_name` | string | Return tasks where tag name contains this string (case-insensitive partial match) |

```bash
# Filter by exact tag ID
curl "https://api.example.com/api/tasks?tag_id=3" \
  -H "Authorization: Bearer <access_token>"

# Filter by partial tag name
curl "https://api.example.com/api/tasks?tag_name=work" \
  -H "Authorization: Bearer <access_token>"
```

### Text search

The `search` parameter performs a case-insensitive partial match against both the task `title` and `description`.

```bash
curl "https://api.example.com/api/tasks?search=documentation" \
  -H "Authorization: Bearer <access_token>"
```

### Overdue filter

Set `overdue=true` to return only tasks whose `due_date` has passed and whose status is not `completed`.

```bash
curl "https://api.example.com/api/tasks?overdue=true" \
  -H "Authorization: Bearer <access_token>"
```

### Date range filters

Filter tasks by their `due_date` using an inclusive range. Both parameters accept ISO 8601 datetime strings.

| Parameter | Description |
|---|---|
| `due_date_from` | Return tasks with `due_date` on or after this datetime |
| `due_date_to` | Return tasks with `due_date` on or before this datetime |

```bash
# Tasks due in Q2 2026
curl "https://api.example.com/api/tasks?due_date_from=2026-04-01T00:00:00&due_date_to=2026-06-30T23:59:59" \
  -H "Authorization: Bearer <access_token>"
```

### Filter by user (admin only)

Admins can scope the results to a specific user's tasks using `user_id`. Non-admins always see only their own tasks regardless of this parameter.

```bash
curl "https://api.example.com/api/tasks?user_id=7" \
  -H "Authorization: Bearer <access_token>"
```

## Combining filters

You can combine multiple filters in a single request. All active filters are applied together (AND logic).

<CodeGroup>
```bash High-priority incomplete tasks due this month
curl "https://api.example.com/api/tasks?priority=high&status=pending&due_date_from=2026-03-01T00:00:00&due_date_to=2026-03-31T23:59:59&sort_by=due_date&sort_order=asc" \
  -H "Authorization: Bearer <access_token>"
```

```bash Search with tag filter, second page
curl "https://api.example.com/api/tasks?search=api&tag_name=docs&page=2&per_page=5" \
  -H "Authorization: Bearer <access_token>"
```

```bash Admin: overdue urgent tasks for a specific user
curl "https://api.example.com/api/tasks?user_id=7&priority=urgent&overdue=true" \
  -H "Authorization: Bearer <access_token>"
```

## Tag listing pagination

`GET /api/tags` supports the same `page`, `per_page`, `sort_by`, and `sort_order` parameters. The default `per_page` for tags is `50` and the default sort is `name` ascending. Supported `sort_by` values for tags: `id`, `name`, `created_at`, `updated_at`.
