TaskForge uses a role-based access model with two roles: `user` and `admin`. Most user management operations are restricted to admins.

## User model

| Field | Type | Description |
|---|---|---|
| `id` | integer | Auto-generated unique identifier |
| `username` | string | Unique. 3–80 characters, alphanumeric with `_` and `-` |
| `email` | string | Unique email address |
| `first_name` | string | Optional first name |
| `last_name` | string | Optional last name |
| `full_name` | string | Computed from `first_name` + `last_name`, falls back to `username` |
| `is_active` | boolean | Whether the account can authenticate |
| `role` | object | Role object with `name` (`admin` or `user`) |
| `created_at` | datetime | ISO 8601 creation timestamp |
| `updated_at` | datetime | ISO 8601 last-updated timestamp |

## Roles

<CardGroup cols={2}>
  <Card title="user" icon="user">
    The default role. Can create, view, update, and delete their own tasks and tags. Cannot access other users' tasks or any admin-only endpoints.
  </Card>
  <Card title="admin" icon="shield">
    Full access. Can view and modify any user's tasks, list all users, delete users, activate or deactivate accounts, and change user roles.
  </Card>
</CardGroup>

## List all users (admin only)

`GET /api/users`

<Warning>
This endpoint requires admin privileges. Non-admin requests return `403 Forbidden`.
</Warning>

<CodeGroup>
```bash Request
curl "https://api.example.com/api/users?page=1&per_page=10&sort_by=created_at&sort_order=desc" \
  -H "Authorization: Bearer <access_token>"
```

```json Response (200)
{
  "success": true,
  "data": {
    "users": [
      {
        "id": 3,
        "username": "jane_doe",
        "email": "jane@example.com",
        "first_name": "Jane",
        "last_name": "Doe",
        "full_name": "Jane Doe",
        "is_active": true,
        "role": { "id": 2, "name": "user", "description": "Standard user" },
        "created_at": "2026-01-15T10:00:00",
        "updated_at": "2026-01-15T10:00:00"
      }
    ],
    "pagination": {
      "page": 1,
      "per_page": 10,
      "total": 1,
      "total_pages": 1,
      "has_next": false,
      "has_prev": false
    }
  }
}
```
</CodeGroup>

Supported `sort_by` fields: `id`, `username`, `email`, `created_at`, `updated_at`.

## Get a user

`GET /api/users/{user_id}`

Users can retrieve their own profile. Admins can retrieve any user's profile.

```bash
curl https://api.example.com/api/users/3 \
  -H "Authorization: Bearer <access_token>"
```

## Update a user profile

`PUT /api/users/{user_id}`

Users can update their own `first_name` and `last_name`. Admins can additionally update `is_active` and `role` for any user.

<CodeGroup>
```bash Self-service profile update
curl -X PUT https://api.example.com/api/users/3 \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "first_name": "Jane",
    "last_name": "Smith"
  }'
```

```bash Admin: change role
curl -X PUT https://api.example.com/api/users/3 \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{"role": "admin"}'
```

```json Response (200)
{
  "success": true,
  "message": "Usuario actualizado con exito",
  "data": {
    "id": 3,
    "username": "jane_doe",
    "email": "jane@example.com",
    "first_name": "Jane",
    "last_name": "Smith",
    "full_name": "Jane Smith",
    "is_active": true,
    "role": { "id": 2, "name": "user", "description": "Standard user" },
    "created_at": "2026-01-15T10:00:00",
    "updated_at": "2026-03-17T11:45:00"
  }
}
```
</CodeGroup>

<Note>
Non-admin users cannot set `role` or `is_active` even if they include those fields — the API silently ignores them.
</Note>

## Deactivate a user (admin only)

`POST /api/users/{user_id}/deactivate`

Deactivated users cannot authenticate. Their tasks and data are preserved.

<Warning>
Admins cannot be deactivated. Attempting to deactivate an admin account returns `403 Forbidden`.
</Warning>

<CodeGroup>
```bash Request
curl -X POST https://api.example.com/api/users/3/deactivate \
  -H "Authorization: Bearer <access_token>"
```

```json Response (200)
{
  "success": true,
  "message": "Usuario desactivado con exito",
  "data": {
    "id": 3,
    "username": "jane_doe",
    "is_active": false,
    "role": { "id": 2, "name": "user", "description": "Standard user" }
  }
}
```
</CodeGroup>

## Activate a user (admin only)

`POST /api/users/{user_id}/activate`

Re-enables a previously deactivated account.

<CodeGroup>
```bash Request
curl -X POST https://api.example.com/api/users/3/activate \
  -H "Authorization: Bearer <access_token>"
```

```json Response (200)
{
  "success": true,
  "message": "Usuario activado con exito",
  "data": {
    "id": 3,
    "username": "jane_doe",
    "is_active": true,
    "role": { "id": 2, "name": "user", "description": "Standard user" }
  }
}
```
</CodeGroup>

## Delete a user (admin only)

`DELETE /api/users/{user_id}`

<Warning>
Deleting a user is permanent and also deletes all of their tasks. Admin accounts cannot be deleted.
</Warning>

<CodeGroup>
```bash Request
curl -X DELETE https://api.example.com/api/users/3 \
  -H "Authorization: Bearer <access_token>"
```

```json Response (200)
{
  "success": true,
  "message": "Usuario eliminado con exito"
}
```
</CodeGroup>

## Permission summary

| Operation | User (self) | User (other) | Admin |
|---|---|---|---|
| Get user profile | Yes | No | Yes |
| Update `first_name` / `last_name` | Yes | No | Yes |
| Update `is_active` / `role` | No | No | Yes |
| List all users | No | No | Yes |
| Activate / deactivate user | No | No | Yes |
| Delete user | No | No | Yes |
